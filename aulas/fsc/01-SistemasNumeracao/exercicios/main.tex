\section*{Parte I -- Aquecimento: Conceitos Fundamentais}
% ══════════════════════════════════════════

\begin{enumerate}

\item Explique, com suas palavras, a diferen\c{c}a entre \emph{n\'{u}mero} e \emph{numeral}. D\^{e} um exemplo de um mesmo n\'{u}mero representado por tr\^{e}s numerais diferentes.

\item Qual a diferen\c{c}a fundamental entre um sistema de numera\c{c}\~{a}o \textbf{posicional} e um \textbf{n\~{a}o-posicional}? Cite um exemplo de cada.

\item No sistema romano (n\~{a}o-posicional), o s\'{i}mbolo \texttt{X} sempre vale~10, independentemente de onde aparece. J\'{a} no sistema decimal, o d\'{i}gito \texttt{3} pode valer~3, 30, 300, etc. Explique por que essa propriedade torna os sistemas posicionais mais eficientes para c\'{a}lculos aritm\'{e}ticos.

\item Quantos s\'{i}mbolos diferentes s\~{a}o usados em cada uma das bases a seguir? Liste-os.
\begin{enumerate}
  \item Base~2 (bin\'{a}rio)
  \item Base~8 (octal)
  \item Base~10 (decimal)
  \item Base~16 (hexadecimal)
\end{enumerate}

\item Complete a frase: ``Com $N$~bits, \'{e} poss\'{i}vel representar no m\'{a}ximo \rule{2cm}{0.4pt}~coisas distintas.'' Justifique.

\end{enumerate}

\section*{Parte II -- Representa\c{c}\~{a}o Polinomial e Convers\~{o}es B\'{a}sicas}
% ══════════════════════════════════════════

\begin{enumerate}[resume]

\item Escreva a \textbf{forma polinomial} de cada n\'{u}mero abaixo e calcule seu valor em decimal:
\begin{multicols}{2}
\begin{enumerate}
  \item $(10110)_2$
  \item $(1101\,0011)_2$
  \item $(537)_8$
  \item $(1A3)_{16}$
  \item $(7F)_{16}$
  \item $(FF)_{16}$
\end{enumerate}
\end{multicols}

\item Converta os seguintes n\'{u}meros \textbf{decimais para bin\'{a}rio}, usando divis\~{o}es sucessivas. Mostre todo o processo.
\begin{multicols}{3}
\begin{enumerate}
  \item $25_{10}$
  \item $100_{10}$
  \item $255_{10}$
  \item $197_{10}$
  \item $1024_{10}$
  \item $2023_{10}$
\end{enumerate}
\end{multicols}

\item Converta os seguintes n\'{u}meros \textbf{decimais para hexadecimal}. Voc\^{e} pode usar o m\'{e}todo que preferir (direto ou passando por bin\'{a}rio).
\begin{multicols}{3}
\begin{enumerate}
  \item $31_{10}$
  \item $200_{10}$
  \item $1000_{10}$
  \item $4095_{10}$
  \item $65535_{10}$
  \item $48879_{10}$
\end{enumerate}
\end{multicols}

\item Converta os seguintes n\'{u}meros \textbf{bin\'{a}rios para hexadecimal}, agrupando em \emph{nibbles}:
\begin{multicols}{2}
\begin{enumerate}
  \item $1011\,0110_2$
  \item $1111\,1111_2$
  \item $10\,1010\,1010_2$
  \item $1\,0000\,0000\,0000_2$
\end{enumerate}
\end{multicols}

\item Converta os seguintes n\'{u}meros \textbf{hexadecimais para bin\'{a}rio}:
\begin{multicols}{3}
\begin{enumerate}
  \item $\texttt{0xCA}$
  \item $\texttt{0xFE}$
  \item $\texttt{0xDEAD}$
  \item $\texttt{0xBEEF}$
  \item $\texttt{0xC0DE}$
  \item $\texttt{0xCAFE}$
\end{enumerate}
\end{multicols}

\begin{dica}
Percebeu algo curioso nos n\'{u}meros hexadecimais acima? Programadores adoram criar ``palavras'' usando d\'{i}gitos hexadecimais (A--F). Esses n\'{u}meros s\~{a}o chamados de \emph{hexspeak}. Alguns exemplos famosos: \texttt{0xDEADBEEF}, \texttt{0xCAFEBABE} (assinatura de arquivos Java \texttt{.class}), \texttt{0xFEEDFACE} (cabe\c{c}alho de bin\'{a}rios Mach-O no macOS).
\end{dica}

\item Realize a seguinte cadeia de convers\~{o}es e verifique que o valor final \'{e} igual ao inicial:
\[
  173_{10} \;\longrightarrow\; (\text{bin\'{a}rio}) \;\longrightarrow\; (\text{hexadecimal}) \;\longrightarrow\; (\text{decimal})
\]

\end{enumerate}

\item Responda:
\begin{enumerate}
  \item Quantos valores distintos um \emph{nibble} (4~bits) pode representar?
  \item Quantos valores distintos um \emph{byte} (8~bits) pode representar?
  \item Quantos d\'{i}gitos hexadecimais s\~{a}o necess\'{a}rios para representar um \emph{byte}?
  \item E para representar uma \emph{word} de 32~bits?
\end{enumerate}

\item Sem fazer c\'{a}lculos, determine rapidamente o valor decimal de:
\begin{enumerate}
  \item $2^{10}$
  \item $2^{16}$
  \item $2^{20}$
  \item $2^{32}$
\end{enumerate}

\begin{dica}
Memorizar as pot\^{e}ncias de~2 \'{e} essencial em computa\c{c}\~{a}o. Lembre: $2^{10} = 1024 \approx 10^3$ (kilo), $2^{20} \approx 10^6$ (mega), $2^{30} \approx 10^9$ (giga).
\end{dica}

\item Um HD \'{e} anunciado como tendo 1~TB (terabyte, $10^{12}$~bytes). Qual o tamanho real em TiB (tebibytes, $2^{40}$~bytes)? Por que existe essa discrep\^{a}ncia entre fabricantes de HD e sistemas operacionais?

\end{enumerate}

\section*{Parte IV -- Bits Representam Qualquer Coisa}
% ══════════════════════════════════════════

\begin{enumerate}[resume]

\item Qual \'{e} o n\'{u}mero m\'{i}nimo de bits necess\'{a}rios para representar \textbf{exclusivamente} os 26~caracteres do alfabeto portugu\^{e}s (sem acentos)?  Quantos caracteres adicionais poderiam ser representados sem bits extras?

\item A tabela ASCII usa 7~bits (arredondados para 1~byte na pr\'{a}tica). Quantos caracteres distintos 7~bits podem representar? Por que isso foi suficiente para o ingl\^{e}s, mas n\~{a}o para o portugu\^{e}s?

\item O padr\~{a}o Unicode utiliza diferentes codifica\c{c}\~{o}es: UTF-8, UTF-16 e UTF-32.
\begin{enumerate}
  \item Quantos caracteres distintos o UTF-32 pode representar \textit{teoricamente}?
  \item Pesquise: quantos \emph{code points} est\~{a}o efetivamente atribu\'{i}dos na vers\~{a}o mais recente do Unicode? O que explica a diferen\c{c}a?
\end{enumerate}

\item C\'{o}digos de cor HTML usam 24~bits (3~bytes), na forma \texttt{\#RRGGBB}, onde cada componente (R, G, B) usa 1~byte.
\begin{enumerate}
  \item Quantas cores distintas essa codifica\c{c}\~{a}o permite?
  \item O branco puro \'{e} \texttt{\#FFFFFF}. Qual \'{e} esse valor em decimal?
  \item O vermelho puro \'{e} \texttt{\#FF0000}. Qual o valor decimal de cada componente (R, G, B)?
  \item Invente uma cor, escreva seu c\'{o}digo hexadecimal e calcule o valor decimal equivalente.
\end{enumerate}

\item Um endere\c{c}o IPv4 usa 32~bits. Um endere\c{c}o IPv6 usa 128~bits.
\begin{enumerate}
  \item Quantos endere\c{c}os IPv4 existem? Expresse como pot\^{e}ncia de~2 e tamb\'{e}m em nota\c{c}\~{a}o decimal aproximada.
  \item Quantos endere\c{c}os IPv6 existem? Expresse como pot\^{e}ncia de~2.
  \item A popula\c{c}\~{a}o mundial \'{e} de aproximadamente $8 \times 10^9$ pessoas. Quantos endere\c{c}os IPv6 cada pessoa poderia ter, em m\'{e}dia? (Apresente o resultado como pot\^{e}ncia de~2 ou de~10.)
\end{enumerate}

\end{enumerate}

\begin{multicols}{2}
\begin{enumerate}
  \item $0101\,0011 + 0010\,1100$
  \item $1100\,0011 + 0011\,1101$
  \item $1111\,0000 + 0001\,0001$
  \item $1011\,1011 + 0111\,0110$
\end{enumerate}
\end{multicols}

\item Realize as seguintes \textbf{subtra\c{c}\~{o}es em bin\'{a}rio} (unsigned, 8~bits):
\begin{multicols}{2}
\begin{enumerate}
  \item $1001\,0110 - 0011\,0101$
  \item $1111\,1111 - 0000\,0001$
  \item $1000\,0000 - 0000\,0001$
  \item $0101\,0101 - 0010\,1010$
\end{enumerate}
\end{multicols}

\item Qual \'{e} o maior n\'{u}mero inteiro sem sinal (\emph{unsigned}) que pode ser representado com:
\begin{enumerate}
  \item 4~bits?
  \item 8~bits?
  \item 16~bits?
  \item 32~bits?
  \item $n$~bits?
\end{enumerate}

\item Um contador bin\'{a}rio de 4~bits come\c{c}a em $0000$ e incrementa de~1 em~1. Escreva a sequ\^{e}ncia completa de valores at\'{e} voltar a $0000$ (ap\'{o}s overflow). Quantos incrementos foram necess\'{a}rios?

\end{enumerate}

\section*{Parte VIII -- Complemento de Dois}
% ══════════════════════════════════════════

\begin{enumerate}[resume]

\item Usando \textbf{complemento de dois com 8~bits}, represente os seguintes valores. Mostre o processo (complementar + somar~1):
\begin{multicols}{3}
\begin{enumerate}
  \item $+23$
  \item $-23$
  \item $+1$
  \item $-1$
  \item $+127$
  \item $-128$
\end{enumerate}
\end{multicols}

\item Converta os seguintes n\'{u}meros em \textbf{complemento de dois para decimal}:
\begin{multicols}{2}
\begin{enumerate}
  \item $0110_2$ \quad (4~bits)
  \item $1101_2$ \quad (4~bits)
  \item $0110\,1111_2$ \quad (8~bits)
  \item $1101\,1011\,0001\,1100_2$ \quad (16~bits)
\end{enumerate}
\end{multicols}

\begin{dica}
Para converter de complemento de dois para decimal, use a f\'{o}rmula: o bit mais significativo tem peso $-2^{N-1}$ e os demais t\^{e}m pesos $+2^{N-2}, \ldots, +2^1, +2^0$.
\end{dica}

\item Usando \textbf{7~bits}, escreva as representa\c{c}\~{o}es de $23$ e $-23$ em:
\begin{enumerate}
  \item Sinal-magnitude
  \item Complemento de dois
\end{enumerate}

\item Em complemento de dois com 12~bits:
\begin{enumerate}
  \item Qual \'{e} o maior n\'{u}mero positivo represent\'{a}vel? (em bin\'{a}rio e decimal)
  \item Qual \'{e} o n\'{u}mero negativo de maior magnitude? (em bin\'{a}rio e decimal)
\end{enumerate}

\item Generalize: em complemento de dois com $n$~bits:
\begin{enumerate}
  \item Qual \'{e} o maior positivo? Expresse como pot\^{e}ncia de~2.
  \item Qual \'{e} o menor negativo? Expresse como pot\^{e}ncia de~2.
  \item Quantos n\'{u}meros positivos, negativos e zeros existem?
\end{enumerate}

\item Expresse o valor $-22$ como inteiro em complemento de dois usando:
\begin{enumerate}
  \item 8~bits
  \item 16~bits
  \item 32~bits
\end{enumerate}
Voc\^{e} percebe um padr\~{a}o? Esse fen\^{o}meno \'{e} chamado \textbf{extens\~{a}o de sinal} (\emph{sign extension}). Descreva-o em uma frase.

\item Explique por que, em complemento de dois com $n$~bits, o valor $-2^{n-1}$ \textbf{n\~{a}o} tem um inverso (positivo) represent\'{a}vel.

\end{enumerate}

\item Os n\'{u}meros a seguir est\~{a}o em complemento de dois com \textbf{4~bits}. Realize as opera\c{c}\~{o}es, converta operandos e resultados para decimal e indique se houve overflow:

\[
  1100_2 + 0110_2
\]
\begin{enumerate}

  \item Interprete como \textbf{signed (complemento de dois)} e verifique se h\'{a} overflow.

\item Um programador armazena a idade de uma pessoa em uma vari\'{a}vel \texttt{int8\_t} (inteiro com sinal de 8~bits em complemento de dois). Qual \'{e} a maior idade que pode ser armazenada? O que aconteceria se ele tentasse armazenar a idade de Jeanne Calment, que viveu 122~anos?

\end{enumerate}

\section*{Parte X -- Problemas Aplicados e Criativos}
% ══════════════════════════════════════════

\begin{enumerate}[resume]

\item \textbf{O Enigma do Odômetro Binário}

\begin{contexto}
Imagine um od\^{o}metro de carro que, em vez de d\'{i}gitos decimais, usa 16~bits bin\'{a}rios (unsigned).
\end{contexto}
\begin{enumerate}
  \item Qual a quilometragem m\'{a}xima que esse od\^{o}metro pode registrar?
  \item Se o carro roda 50~km por dia, ap\'{o}s quantos dias o od\^{o}metro ``zera'' (overflow)?
  \item Se o od\^{o}metro usasse 32~bits, quantos anos levariam para zerar, nas mesmas condi\c{c}\~{o}es?
\end{enumerate}

\item \textbf{Mensagem Secreta}

\begin{contexto}
Uma mensagem foi codificada convertendo cada letra para sua posi\c{c}\~{a}o no alfabeto (A=1, B=2, \ldots, Z=26) e depois para bin\'{a}rio com 5~bits.
\end{contexto}

Decodifique: \texttt{01000 01001 01110 00001 10010 01001 01111}

\item \textbf{O Bug do Pac-Man}

\begin{contexto}
No jogo original \emph{Pac-Man} (1980), o contador de n\'{i}vel era armazenado em um \'{u}nico byte (8~bits, unsigned). Quando o jogador chegava ao n\'{i}vel~256, ocorria um famoso \emph{bug}.
\end{contexto}
\begin{enumerate}
  \item Explique, do ponto de vista de sistemas de numera\c{c}\~{a}o, o que aconteceu no n\'{i}vel~256.
  \item Qual seria o maior n\'{i}vel poss\'{i}vel se o contador usasse 16~bits?
  \item E se usasse um \texttt{int8\_t} (8~bits com sinal em complemento de dois)?
\end{enumerate}

\item \textbf{Nuclear Gandhi}

\begin{contexto}
Na s\'{e}rie de jogos \emph{Civilization}, existe uma lenda urbana de que o l\'{i}der Gandhi, normalmente pac\'{i}fico, se tornava extremamente agressivo. A ``agressividade'' de Gandhi come\c{c}ava em~1 (de~0 a~255, armazenada em 1~byte unsigned). Ao adotar a democracia, a agressividade era reduzida em~2.
\end{contexto}
\begin{enumerate}
  \item Calcule $1 - 2$ em aritm\'{e}tica de 8~bits unsigned. Qual valor resultaria?
  \item Explique o comportamento inesperado usando o conceito de overflow/underflow.
  \item Se a vari\'{a}vel fosse do tipo \texttt{int8\_t} (signed), qual seria o resultado de $1-2$? O bug aconteceria?
\end{enumerate}

\item \textbf{O Bug do Ano 2038}

\begin{contexto}
Muitos sistemas Unix armazenam timestamps como o n\'{u}mero de segundos desde 1\textordmasculine{} de janeiro de 1970 (``\emph{Unix epoch}''), usando um inteiro com sinal de 32~bits (complemento de dois).
\end{contexto}
\begin{enumerate}
  \item Qual \'{e} o maior valor positivo que um \texttt{int32\_t} pode armazenar?
  \item Quantos segundos h\'{a} em um ano (considere 365,25 dias)?
  \item Em que ano aproximado o contador ir\'{a} ``estourar'' (overflow)?
  \item Qual \'{e} a solu\c{c}\~{a}o adotada pelos sistemas modernos?
\end{enumerate}

\item \textbf{Endere\c{c}os de Mem\'{o}ria}

\begin{contexto}
Um processador usa endere\c{c}os de 32~bits para acessar a mem\'{o}ria RAM.
\end{contexto}
\begin{enumerate}
  \item Quantos bytes de mem\'{o}ria podem ser endere\c{c}ados?
  \item Converta essa quantidade para GiB ($2^{30}$~bytes).
  \item Explique por que sistemas operacionais de 32~bits t\^{e}m um limite pr\'{a}tico de~4~GiB de RAM.
  \item Processadores de 64~bits resolvem esse problema. Quantos bytes poderiam, em teoria, ser endere\c{c}ados? \'{E} um valor pr\'{a}tico?
\end{enumerate}

\item \textbf{Placas de Ve\'{i}culos}

\begin{contexto}
As placas de ve\'{i}culos no Brasil (padr\~{a}o Mercosul) t\^{e}m o formato \texttt{LLLNLNN}, onde L~=~letra (26~possibilidades) e N~=~d\'{i}gito (10~possibilidades).
\end{contexto}

\item \textbf{Criptografia e Bases}

A codifica\c{c}\~{a}o \emph{Base64} \'{e} amplamente usada na internet para transmitir dados bin\'{a}rios como texto. Ela usa 64~s\'{i}mbolos para representar dados.
\end{contexto}

\end{enumerate}
\end{multicols}

\item Converta os seguintes n\'{u}meros \textbf{octais para decimal}:
\begin{multicols}{3}
\begin{enumerate}
  \item $(77)_8$
  \item $(377)_8$
  \item $(1750)_8$
  \item $(644)_8$
  \item $(755)_8$
  \item $(2000)_8$
\end{enumerate}

\item A convers\~{a}o entre \textbf{bin\'{a}rio e octal} \'{e} direta, agrupando os bits de 3~em~3 (da direita para a esquerda). Converta:
\begin{multicols}{2}
\begin{enumerate}
  \item $1101\,0110_2 \longrightarrow \text{octal}$
  \item $1\,0000\,0000_2 \longrightarrow \text{octal}$
  \item $(375)_8 \longrightarrow \text{bin\'{a}rio}$
  \item $(17)_8 \longrightarrow \text{bin\'{a}rio}$
\end{enumerate}

\begin{dica}

\end{dica}

\item \textbf{Permiss\~{o}es Unix}

\begin{contexto}
No Linux, cada arquivo tem permiss\~{o}es para tr\^{e}s categorias: dono, grupo e outros. Cada categoria tem 3~bits: leitura (r=4), escrita (w=2) e execu\c{c}\~{a}o (x=1). A soma desses valores forma um d\'{i}gito octal.
\end{contexto}
\begin{enumerate}
  \item Qual \'{e} a representa\c{c}\~{a}o bin\'{a}ria (9~bits) e octal das permiss\~{o}es \texttt{rwxr-xr--}?
  \item Um arquivo tem permiss\~{a}o octal \texttt{640}. Quais s\~{a}o as permiss\~{o}es de cada categoria?
  \item Por que o sistema octal \'{e} particularmente conveniente para permiss\~{o}es Unix? (Pense na rela\c{c}\~{a}o entre 3~bits e 1~d\'{i}gito octal.)
\end{enumerate}

\end{enumerate}

\begin{contexto}

\begin{contexto}
Al\'{e}m de opera\c{c}\~{o}es aritm\'{e}ticas, computadores realizam opera\c{c}\~{o}es \textbf{l\'{o}gicas} bit a bit: AND (\&), OR ($|$), XOR ($\oplus$) e NOT ($\sim$). Cada opera\c{c}\~{a}o \'{e} aplicada individualmente a cada par de bits correspondentes.

  \item Uma propriedade do XOR: $A \oplus B \oplus B = A$. Explique por que essa propriedade \'{e} \'{u}til em criptografia simples e em algoritmos de troca de vari\'{a}veis sem usar uma vari\'{a}vel tempor\'{a}ria.

  \item $A \leftarrow A \oplus B$

Mostre o valor de $A$ e $B$ ap\'{o}s cada passo. O que aconteceu?

Em 2014, o clipe de ``Gangnam Style'' do PSY ultrapassou 2.147.483.647 visualiza\c{c}\~{o}es no YouTube. O contador do YouTube parou de funcionar.

Cada placa de rede possui um endere\c{c}o MAC \'{u}nico de 48~bits, geralmente escrito como 6~pares hexadecimais: \texttt{AA:BB:CC:DD:EE:FF}.

Um c\'{o}digo de barras EAN-13 (usado em produtos) codifica 13~d\'{i}gitos decimais.

\item \textbf{GPS e Precis\~{a}o}

\begin{contexto}
Coordenadas GPS s\~{a}o freq\"{u}entemente armazenadas como n\'{u}meros inteiros com sinal de 32~bits, representando micrograus (milion\'{e}simos de grau). Por exemplo, a latitude $-29{,}951\,066\,\!^\circ$ seria armazenada como $-29\,951\,066$.

Uma das cifras mais simples em criptografia \'{e} o XOR com uma chave.

Uma mensagem de 1~byte \texttt{M = 0x48} \'{e} cifrada com a chave \texttt{K = 0x5A} usando XOR.

  \item Mostre que aplicar XOR novamente com a mesma chave recupera a mensagem: $M = C \oplus K$.

  \item Por que a cifra XOR sozinha n\~{a}o \'{e} segura para criptografia real?

\section*{Parte XI -- Desafios (Opcional)}
% ══════════════════════════════════════════

\begin{enumerate}[resume]

\begin{desafio}

\item Demonstre algebricamente que, em complemento de dois com $n$~bits, o inverso aditivo de um n\'{u}mero $x$ pode ser obtido complementando todos os bits e somando~1. Ou seja, prove que:
\[
  -x = \overline{x} + 1
\]
onde $\overline{x}$ denota o complemento bit a bit de~$x$.

\textit{Dica: comece observando que $x + \overline{x} = 2^n - 1$ (todos os bits em~1).}

\item \textbf{Sistema de base~$-2$ (``negabin\'{a}rio'')}

Sim, \'{e} poss\'{i}vel usar bases negativas! Na base~$-2$, os pesos das posi\c{c}\~{o}es s\~{a}o $(-2)^0 = 1$, $(-2)^1 = -2$, $(-2)^2 = 4$, $(-2)^3 = -8$, etc.

\begin{enumerate}
  \item Calcule o valor decimal de $11010_{-2}$.
  \item Represente $-6$ na base~$-2$.
  \item Qual \'{e} a vantagem te\'{o}rica desse sistema em rela\c{c}\~{a}o ao complemento de dois?
\end{enumerate}

\item \textbf{BCD (\emph{Binary-Coded Decimal})}

Na codifica\c{c}\~{a}o BCD, cada d\'{i}gito decimal \'{e} representado individualmente por 4~bits.

\begin{enumerate}
  \item Represente $9742_{10}$ em BCD.
  \item O BCD \'{e} mais ou menos eficiente que bin\'{a}rio puro em termos de espa\c{c}o? Justifique comparando quantos bits s\~{a}o necess\'{a}rios para representar o n\'{u}mero $255$ em ambos os formatos.
  \item Pesquise: em que tipo de aplica\c{c}\~{a}o o BCD ainda \'{e} utilizado e por qu\^{e}?
\end{enumerate}

\item \textbf{O Curioso Caso do $-0$ no Espa\c{c}o}

Em 1996, o foguete Ariane~5 da Ag\^{e}ncia Espacial Europeia explodiu 37~segundos ap\'{o}s o lan\c{c}amento. A causa raiz foi a convers\~{a}o de um n\'{u}mero de ponto flutuante de 64~bits para um inteiro com sinal de 16~bits, gerando overflow.

\begin{enumerate}
  \item Qual \'{e} o intervalo de valores de um inteiro com sinal de 16~bits (complemento de dois)?
  \item Se a velocidade horizontal do foguete era armazenada como um valor que excedia $32\,767$, o que aconteceu na convers\~{a}o?
  \item Que li\c{c}\~{a}o sobre sistemas de numera\c{c}\~{a}o e overflow esse acidente ensina?
\end{enumerate}

\item \textbf{Soma de Verifica\c{c}\~{a}o (\emph{Checksum})}

Uma t\'{e}cnica simples de verifica\c{c}\~{a}o de integridade de dados consiste em somar todos os bytes de uma mensagem (em complemento de um) e usar o resultado como \emph{checksum}.

Considere a mensagem formada pelos bytes: \texttt{0x4F}, \texttt{0x6C}, \texttt{0x61}.
\begin{enumerate}
  \item Some os tr\^{e}s bytes em bin\'{a}rio (8~bits). Se houver \emph{carry}, some-o de volta ao resultado.
  \item Complemente (inverta) todos os bits. Este \'{e} o checksum.
  \item Ao receber a mensagem junto com o checksum, como o receptor pode verificar a integridade?
\end{enumerate}

\item \textbf{O Sistema Tern\'{a}rio Balanceado}

O computador sovi\'{e}tico Setun (1958) usava o sistema \textbf{tern\'{a}rio balanceado}, com d\'{i}gitos $\{-1, 0, +1\}$ (representados como $T$, $0$, $1$). As posi\c{c}\~{o}es valem pot\^{e}ncias de~3.

\begin{enumerate}
  \item Calcule o valor decimal de $1T01$ (onde $T = -1$).\\
  \textit{Dica: $1 \cdot 3^3 + (-1) \cdot 3^2 + 0 \cdot 3^1 + 1 \cdot 3^0$}
  \item Represente $-13$ em tern\'{a}rio balanceado.
  \item Qual vantagem esse sistema tem sobre complemento de dois para representar n\'{u}meros negativos?
\end{enumerate}

\item \textbf{N\'{u}meros de Ponto Flutuante -- Introdu\c{c}\~{a}o}

O padr\~{a}o IEEE~754 (precis\~{a}o simples, 32~bits) divide os bits em tr\^{e}s campos:
\begin{itemize}
  \item 1~bit de sinal ($s$)
  \item 8~bits de expoente ($e$), com \emph{bias} de 127
  \item 23~bits de mantissa ($m$), com um ``1'' impl\'{i}cito
\end{itemize}
O valor \'{e}: $(-1)^s \times 1{,}m \times 2^{(e - 127)}$

\begin{enumerate}
  \item O n\'{u}mero $\texttt{0 10000010 10100000000000000000000}$ est\'{a} em IEEE~754. Qual \'{e} seu valor decimal?
  \item Represente $-6{,}5$ em IEEE~754 (32~bits).
  \item Por que n\'{u}meros de ponto flutuante podem representar valores muito maiores que inteiros de 32~bits, mas com menos precis\~{a}o?
\end{enumerate}

\item \textbf{C\'{o}digo de Gray}

O C\'{o}digo de Gray \'{e} um sistema bin\'{a}rio onde dois valores consecutivos diferem em apenas 1~bit. Ele \'{e} usado em encoders rotat\'{o}rios e na corre\c{c}\~{a}o de erros.

A convers\~{a}o de bin\'{a}rio para Gray \'{e}: $G_i = B_i \oplus B_{i+1}$ (XOR entre bits adjacentes, onde $B_{n}=0$ para o bit mais significativo).

\begin{enumerate}
  \item Construa a tabela do C\'{o}digo de Gray para 4~bits (0 a 15).
  \item Verifique que valores consecutivos diferem em exatamente 1~bit.
  \item Converta o bin\'{a}rio $1011$ para C\'{o}digo de Gray.
  \item Por que o C\'{o}digo de Gray \'{e} prefer\'{i}vel ao bin\'{a}rio natural em encoders rotat\'{o}rios? (Pense no que acontece quando m\'{u}ltiplos bits mudam simultaneamente.)
\end{enumerate}

\item \textbf{Paradoxo do Anivers\'{a}rio em Hashes}

\begin{contexto}
Fun\c{c}\~{o}es hash mapeiam dados de tamanho arbitr\'{a}rio para um valor de tamanho fixo. O ``paradoxo do anivers\'{a}rio'' implica que colis\~{o}es (dois inputs com o mesmo hash) s\~{a}o prov\'{a}veis ap\'{o}s aproximadamente $\sqrt{2^n}$ tentativas, onde $n$~\'{e} o n\'{u}mero de bits do hash.
\end{contexto}
\begin{enumerate}
  \item Para um hash de 32~bits, ap\'{o}s quantas tentativas uma colis\~{a}o se torna prov\'{a}vel? Expresse como pot\^{e}ncia de~2.
  \item Para um hash de 256~bits (SHA-256), calcule o mesmo valor.
  \item A rede Bitcoin processa $\sim 10^{20}$ hashes por ano. Quantos anos seriam necess\'{a}rios para encontrar uma colis\~{a}o em SHA-256?
\end{enumerate}

\end{desafio}

\end{enumerate}
